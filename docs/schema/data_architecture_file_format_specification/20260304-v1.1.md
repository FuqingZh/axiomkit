# Data Architecture & File Format Specification v1.1

- Date: `2026-03-04`
- Scope: 通用 step-based 数据处理与分析流水线（非领域绑定）

## 1. 角色分层

本规范采用四层职责分离：

1. 数据真源（Data Source of Truth）：`Parquet / Parquet Dataset`
2. 可读交换层（Human-Readable Exchange）：`TSV`
3. 语义配置真源（Semantic Source of Truth）：`TOML`
4. 入口与交接摘要（Manifest & Handoff）：`meta/meta.json`

## 2. 格式选择与职责

### 2.1 Parquet（主数据）

适用：

1. 中大型结构化表（计算、聚合、过滤、连接）。
2. 需要稳定 schema 与跨语言一致读写。

约束：

1. 作为分析主数据输入/输出时，字段类型必须稳定。
2. 不存放配置参数（参数在 TOML）。
3. 单逻辑表默认落地为 `data-*.parquet` 单文件。
4. 仅当主数据需要按分区键裁剪读取、增量写出或天然按组拆分时，才落地为 `data-*/` Parquet dataset。
5. 使用 Parquet dataset 时，必须在架构文档与 `meta/meta.json` 中显式声明分区键。

### 2.2 TSV（可读交换）

适用：

1. 人工检查。
2. 绘图输入。
3. 外部轻量交换。

约束：

1. TSV 不作为主分析真源。
2. 大规模计算优先读取 Parquet。

### 2.3 TOML（配置真源）

适用：

1. 流程参数、阈值、规则、执行选项。
2. 跨阶段统一配置和版本化审计。

约束：

1. 不使用 YAML 作为配置真源。
2. 不把运行时统计写入 TOML。

### 2.4 meta/meta.json（交接入口）

适用：

1. 上下游模块交接入口。
2. 记录运行摘要、关键产物索引、实际生效配置快照。

约束：

1. 不在 `meta.json` 定义业务语义真源。
2. `meta.json` 只做入口索引与运行摘要。

## 3. 配置来源优先级

推荐统一优先级：

1. CLI 显式参数
2. 阶段 TOML 配置
3. 上游 `meta.json` 推断
4. 内置默认值

## 4. 文件命名规范（强制）

### 4.1 前缀类别

输出文件前缀统一采用：

1. `data-`：结构化数据产物（表、矩阵、映射）
2. `info-`：说明类信息（摘要、清单、映射索引）
3. `graph-`：图形产物（png/pdf/svg）

可选补充：

1. `meta-`：元信息扩展文件
2. `log-`：日志归档文件
3. `conf-`：导出的生效配置快照
4. `bundle-`：打包归档产物（zip/tar.zst 等）
5. `frontend-`：前端直连消费产物（json/tsv/任意静态资源）

约束说明：

1. `bundle-` 与 `frontend-` 属于可选保留前缀。
2. `bundle-` 与 `frontend-` 不作为数据真源输入。
3. `frontend-*` 必须可由 `data-/info-/graph-` 重新生成。

### 4.2 分隔符语义

1. `-`：类别与语义段分隔
2. `_`：词组内部连接

示例：

1. `data-quant_summary.parquet`
2. `info-run_manifest.tsv`
3. `graph-nine_quadrant.png`
4. `data-identifier_map.tsv`
5. `bundle-report_assets.tar.zst`
6. `frontend-graph_manifest.json`

## 5. 反模式（禁止）

1. 用 TSV 替代 Parquet 作为主分析输入。
2. 把配置参数硬编码在脚本且不落 TOML。
3. 在 `meta.json` 中定义会改变计算行为的参数。
4. 输出文件命名无前缀、无语义、不可审计。
5. 把 `bundle-*` 或 `frontend-*` 当作跨模块计算真源。

## 6. 最小落地示例

```text
<stage_root>/
├── configs/
│   └── stage.toml
├── out/
│   ├── data-*.parquet
│   ├── data-*/
│   ├── info-*.tsv
│   ├── graph-*.png
│   ├── bundle-*.tar.zst
│   └── frontend-*.json
└── meta/
    └── meta.json
```
